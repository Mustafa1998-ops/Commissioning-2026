# Day 14: The SOC Commander — Building a SOC Playbook

## Overview
This project focuses on incident response consistency by designing, reviewing, and automating SOC playbooks. It outlines the process of turning ad-hoc responses into structured, repeatable plans and maps these procedures to Security Orchestration, Automation, and Response (SOAR) platforms.

## Key Concepts & Skills Demonstrated
* **Playbook Development:** Built step-by-step incident response playbooks using a standard 10-section template.
* **Incident Response Phases:** Structured responses encompassing Detection, Triage, Containment, Eradication, Recovery, and Lessons Learned.
* **Threat Scenarios Addressed:** Created dedicated playbooks for RDP Brute Force attacks and Malware Download incidents (e.g., PowerShell download cradles).
* **Peer Review & Versioning:** Participated in playbook peer reviews to identify missing steps (such as endpoint isolation) and published version-controlled updates (e.g., pushing from v1.0 to v1.1).
* **SOAR Integration:** Mapped incident response procedures into a SOAR flowchart.
* **Automation Assessment:** Evaluated which phases of the response lifecycle can be fully automated, semi-automated, or must remain manual tasks.

## Completed Tasks
* **Tasks 14.2 - 14.4:** Used the SOC Playbook Builder to draft comprehensive response protocols for RDP brute force activity and malware payloads.
* **Task 14.5:** Executed a Playbook Review Simulator to leave actionable comments and suggest containment/recovery improvements for peer playbooks.
* **Task 14.6:** Approved and published a finalized playbook via a Version Tracker system.
* **AT54 (SOAR):** Configured a SOAR integration flowchart, assigning manual or automated execution parameters to the Detection, Triage, Containment, Eradication, and Recovery phases.
